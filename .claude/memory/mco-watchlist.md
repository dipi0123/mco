분기마다 이 목록을 점검한다. 흩어진 출처(사업성 §8·경쟁지형 §6·[[mco-viability-verdict]]·[[mco-competitive-landscape]]·차터 §9)의 **단일 취합 진입점**.

**킬 트리거 (발동 시 논거 재점검·피벗 검토):**
1. **정밀도 실험 실패** — 게이트·선택 정밀도가 최강 염가 베이스라인(full-context·BM25) +10점을 못 넘음 → 코어 차별점 붕괴.
2. **OpenAI Memory API 출시·개방** — 크로스 툴 메모리 API가 열리면 이동성 논거 재점검.
3. **Nous 등 메모리 자체 번들** — 주요 플레이어가 메모리를 자체 번들.

**감시 항목 (분기 재확인):**
- OpenAI 메모리 API 출시 여부 · Codex Memories 클라우드 동기화 여부
- Anthropic Managed Agents 메모리의 외부(비-Anthropic) 개방 여부
- AWS Bedrock AgentCore 메모리의 벤더 횡단 개방 여부
- mem0 OpenMemory 권한 스코프 추가 여부 · Engram 개인/크로스툴 확장 여부
- 메모리 오픈 표준: AAIF 메모리 프로젝트 / W3C AI Agent Memory Interoperability CG 발족 여부
- Hermes Cloud 자체 메모리 번들 여부 · Aristo(Membase, 국내) 행보
- 플랫폼 문(門) 정책: ChatGPT Apps 심사(데이터 최소화 vs 기억 제품) · Claude 무료 커넥터 정책(현재 무료 1슬롯)

Why: 감시/킬 항목이 5곳에 흩어져 분기 리뷰 때 어느 목록이 정본인지 모호 → 단일 취합.

How to apply: 분기 리뷰의 단일 진입점. 트리거 발동 시 해당 논거(차터 §2·§9·[[mco-viability-verdict]]) 재점검. 항목 변동 시 이 파일만 갱신, 출처 문서는 상세 근거로 유지. [[mco-competitive-landscape]] [[mco-viability-verdict]]
