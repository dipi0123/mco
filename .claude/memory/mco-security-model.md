**전제 — 구조 리스크.** MCO는 lethal trifecta(① 비공개 데이터 접근 ② 비신뢰 콘텐츠 노출 ③ 외부 통신)의 세 변을 제품 구조로 전부 갖는다: 최고 민감의 지속 메모리를 보유하고(①), 함대 write-back이 에이전트 런 산출물 — 비신뢰 웹·문서·도구 출력 유래 — 을 기억으로 흡수하며(②), 그 기억을 외부 통신 가능한 임의 에이전트에 주입한다(③). LLM은 운영자 지시와 콘텐츠에 심긴 주입 지시를 구분하지 못하므로, 방어 본체는 **아키텍처 제약**("주입된 입력이 중대 행동을 트리거할 수 없게")이고 스캔·필터는 보조다 — 가드레일 95% 차단은 앱보안 기준 낙제(Willison).

**통제 6항:**

1. **기억 신뢰 티어링** — provenance에 trust level 필드를 추가한다(사람 단언 > 검증된 런 산출 > 비신뢰 콘텐츠 유래). 저신뢰 기억은 고권한 문맥(교차 스코프 브리핑·자격증명 인접 작업)에 자동 주입 금지. 강제 지점 = 기존 가치 판단 게이트·기억 선택(역할 ①⑤) — 보안이 '저장<선택' 코어 메커니즘 위에 얹히므로 신규 컴포넌트가 필요 없다. → 스키마 v0.1 필드 + 차터 §9 보안 항목(v0.3 부분확정).
2. **write-back = 중대 행동** — 스코프 X의 런은 스코프 X에만 write. 교차 스코프·교차 테넌트 write는 예외 없이 사람 승인 게이트. RLS의 read 격리와 대칭을 이루며, 오염 기억이 타 계정·법인·프로젝트로 전파되는 confused-deputy 경로를 봉쇄한다. → 스키마 v0.1 + 차터 §9 보안 항목(v0.3 부분확정).
3. **스캔 = 보조 원칙 명문화** — '주입 전 스캔'(인프라 §6, Hermes 차용)은 유지하되 방어 본체가 아님을 보안 모델에 명기한다. 본체는 통제 1·2. → 차터 §9 보안 항목(v0.3 부분확정).
4. **대시보드 = IAP 게이트** — 예외 인박스·감사·권한 대시보드에 인증을 자체 빌드하지 않는다('인증은 빌드하지 않는다' 원칙의 연장). identity-aware proxy 패턴(IdP + 정책 엔진 + 리버스 프록시 — Cloudflare Access류/Pomerium) + SSO로 앞단 차단. → 인프라 §9 #5 대시보드 스택 결정의 입력.
5. **신뢰 헤더 규율** — forwarded 계열 헤더(CF-Connecting-IP 등)는 엣지가 세팅한 것만 신뢰하고, 클라이언트 제공 신원 헤더는 경계에서 스트립한다. trusted-proxy 패턴의 표준 함정(신원 헤더 스푸핑 = 인증 전체 무력화, 프록시 = 단일 실패점) 대응. → 구현 1일차 체크리스트.
6. **제로 상태 인증 테스트** — 온보딩·OAuth·첫 5분 경로는 기연결 개발 계정이 아니라 신규 미연결 계정으로 검증한다(기연결 상태 테스트가 미페어링 경로 버그를 가리는 실증 사례 존재). → 인프라 §8 '연결 직후 self-test'에 통합.

**근거:** Simon Willison 'The lethal trifecta for AI agents'(2025-06) · Pomerium 'Zero Trust Access for MCP' · OpenClaw trusted-proxy 인증 모드(#1560) 사례 분석.

Why: 기억 제품은 lethal trifecta의 정중앙이라 보안 모델 미결(§9 보안 항목(v0.3 부분확정)) 상태로 구현에 들어갈 수 없다. 통제 1·2는 코어 차별점(게이트·정밀 선택·강제 스코프)과 같은 메커니즘 위에 있어 보안이 곧 제품 논거가 된다.

How to apply: 차터 v0.3 · 스키마 v0.1 · 대시보드 스택 결정 · MVP 스펙에서 이 6항을 확정 입력으로 사용. 상세 프라이버시 항목(암호화 수준·삭제권 보증·보존 정책)은 여전히 §9 보안 항목(v0.3 부분확정) 미결 — 이 문서는 그 중 아키텍처 통제 부분을 선확정한 것. **문서 반영 완료(2026-07-12)**: 인프라구축 §6(전제+통제 1~5)·§8(제로 상태)·§9 #5·#6 / 아키텍처 정본 §6 보안·§9 미결 ⓑ·개정 이력 / 아키텍처설명 §3 trust_level 필드·§5 write-back 통제 — 차터 반영은 v0.3 때.
