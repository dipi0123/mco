> **⚠️ 2026-07-12 정합 감사: 이 '구조화 3층'은 구식이다.** 정본 스키마는 `Product/MCO_아키텍처_v0_1.md` §4의 **type 6종 × 인출 2계층(설정성/사건성)**으로 대체됐다(charter §4가 3층 흡수 명시). 아래 3층은 도출-출처 관점의 초기 씨앗으로만 참고하고, 스키마·DB 설계는 정본 2축에서 시작할 것.

브레인스톰 단계 구상 (현재 합의, 상세 스키마 미설계):

- **수집** — 입력 데이터 중 지속적으로 쓸 만한 기억(나에 대한 정보)을 따로 리스트업.
- **구조화 3층** — ① **명시적으로 저장된 정보**(사용자가 직접 준 사실) ② **맥락에서 추출한 의미**(대화 맥락에서 뽑은 함의) ③ **사건의 조각들**(에피소드 단편).
- **선택** — 질문이 들어오면 필요한 기억만 인출. 여기가 코어 — [[mco-selection-over-storage]].

**Why:** 저장 스키마와 인출 전략 설계의 출발점. 3층 구분이 스몰 모델의 '저장 범위 결정'과 '충돌 감지'의 기반 축이 된다.

**How to apply:** 메모리 스키마·DB 설계 시 이 3층을 1차 분류 축으로 시작하고, 구조가 바뀌면 이 메모리를 갱신한다. [[mco-concept-core]].
