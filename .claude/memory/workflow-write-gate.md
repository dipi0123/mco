Claim OS 프로젝트에서 검증된 Ethan의 협업 워크플로를 MCO에도 적용한다 (2026-07-09 하네스 구성 시 이식):

- **"작성" 게이트** — 큰 결정·산출물은 채팅에서 먼저 정렬하고, Ethan이 **"작성"** 으로 트리거할 때 생성한다. 정렬 없이 임의로 문서·구조를 선행 생성하지 않는다.
- **현재 합의 원칙** — 브레인스톰 단계라 '잠긴 설계'는 아직 없다. 메모리·문서는 논의용 참조이지 권위가 아니다 — 결정은 merit로 논증하고, 결정이 서면 메모리를 그에 맞춰 갱신한다.
- **정본 용어 존중** — 명칭(MCO)·핵심 표현(Personal Memory Passport·제조 메모리 컨텍스트 등)은 Ethan이 확정한 것을 임의 개명하지 않는다. 바꾸려면 채팅에서 제안·정렬 먼저.
- **브레인스톰 캡처** — 세션에서 나온 새 합의·기각은 그 자리에서 `.claude/memory/`에 기록한다(기각도 기록 — 같은 논의 반복 방지).

**Why:** 임의 개명·재구조화는 다른 프로젝트에서 강한 교정을 받은 전력이 있다. 여러 프로젝트를 병행하는 Ethan에게는 프로젝트 간 통일된 협업 리듬 자체가 UX의 핵심이다.

**How to apply:** 세션 시작 시 MEMORY.md 로드 → 논의 → 합의를 메모리로 캡처 → "작성" 트리거에만 산출물 생성 → 산출물·결정 변경 시 메모리·차터 갱신.
