2026-07-11 합의 — v0.2 웨지("하네스 플러그인 우선, 코딩 파워유저 먼저")의 재개정. 기존 판단은 **기각이 아니라 병행 구조로 개정** (기각 사유 기록: 하드코어 로컬 설치 UX로는 사용자 확보 불가 — Ethan 지시 + 채널 실측).

- **채널 실측(2026-07)**: Claude 커넥터 = **무료 플랜 포함** 웹·데스크톱·모바일 동일 동작, 단 무료는 커스텀 커넥터 1개 제한("그 한 슬롯을 이기는 게임", 첫 5분이 생존 장치). ChatGPT = 커스텀 MCP가 dev mode 전용(공식 문서 "exclusively for developers") + Lockdown Mode 리스크 → 일반인 경로는 Apps 디렉토리 심사뿐. Gemini = 서드파티 주입 경로 없음. → **Claude-first는 취향이 아니라 소거법.**
- **구조**: 웹 = 획득·일반용 표면 / 하네스(Claude Code·Cursor)·Hermes 프로바이더 = 심화·파워용 표면(자동 주입 제어 가능) / 코어 MCP 서버 하나에 표면별 얇은 어댑터.
- **웹 표면의 구조적 한계(정직하게)**: 서드파티 기억은 모델이 도구를 불러야 작동(자동 주입 아님) → 첫 경험은 '저절로 아는'이 아니라 '연결하자마자 보여주는'(웜스타트 스캔·부패도 검진 = 우리가 제어하는 호출로 100% 재현). 온보딩에서 사용자 지침 1줄 세팅 안내.
- **v1 마케팅 금지어**: "GPT에서도 딸깍"(현재 지킬 수 없는 약속).

Why: Ethan 방향 지시("일반 클로드·GPT 웹에서 MCP 플러그인 딸깍, 인프라 설정 없이 어디서든 통일") + 플랫폼 문(門) 정책 실측이 일치.

How to apply: MVP 스펙에서 1차 표면 = Claude 커넥터 + 웹 대시보드, 병행 = Claude Code 플러그인·Hermes 프로바이더. 차터 §7 v0.3 갱신 시 이 내용 반영. 감시: Claude 무료 커넥터 정책 변화, ChatGPT Apps 심사 정책. [[mco-mvp-direction]] [[mco-architecture-v01]]
