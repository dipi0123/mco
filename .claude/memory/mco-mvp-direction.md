2026-07-10 웨지 스왑 → **2026-07-12 v0.3 재개정: 웹 커넥터 우선([[mco-wedge-web-connector]])** (v0.1: MCP 서버 + 브라우저 확장 + 대시보드 / 개발자·컨설턴트·변호사):

- **MVP 구성(v0.3):** ① **Claude 커넥터 + 웹 대시보드** — 1차 표면(획득·일반용) ② **MCP 메모리 서버** — 코어 ③ **하네스(Claude Code·Cursor)·Hermes** — 병행 심화 표면 ④ **결정 인박스형 대시보드** — 충돌·승격/만료·출처 감사만 올리는 예외 처리함(CRUD UI 아님). 브라우저 확장은 후순위.
- **초기 페르소나:** 컨텍스트가 돈인 프로슈머(멀티 법인·클라이언트·서류·마감). 코딩 에이전트 파워유저 = 폐기 아니라 **경로 A(슬롯 정밀도)의 초기 획득 채널**. GTM = 경로 A·B 병렬(B = 서류 컴포저).
- **최우선 기술 검증:** write-back 정밀도 — 코드 이전에 실제 에이전트 세션 로그 + 역할 ① 트리아지 프롬프트로 가능. precision 수동 채점.
- 인증/소유: OAuth 계정 바인딩 — [[mco-positioning-bigtech-gap]].

**Why:** 함대 리프레임 [[mco-fleet-reframe]] + 경쟁 지형 [[mco-competitive-landscape]] 결론 — 이동성 웨지(확장)는 혼잡 시장, 하네스/함대 웨지는 공백. 코딩 에이전트 유저가 크로스 런타임 맥락 단절 페인을 가장 절실히, 지금 느낀다.

**How to apply:** MVP 스펙 작업 시 이 3요소에서 출발하되 write-back 정밀도 검증을 스펙보다 먼저. 구성·타깃이 바뀌면 이 메모리를 갱신. [[mco-concept-core]]
