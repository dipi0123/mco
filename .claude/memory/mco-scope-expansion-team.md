2026-07-15 정렬 (현재 합의). 계기: 퇴근길 AI 영상("AI 1인 창업, 이젠 짜증만 납니다" — 구글드라이브 공유 폴더로 팀+에이전트 컨텍스트 공유) 리뷰. 파일 기반 공유 메모리 = 나이브 선행 사례이자 수요 증거.

**결정:** MVP는 **개인 AI 에이전트 메모리**를 기본으로 관리(현 웨지 [[mco-mvp-direction]] 유지). 이후 옵트인으로 **팀/공유 프로젝트 스코프**로 확장 — "팀으로 생성"이 스코프를 개인→공유로 승격. 컨셉 포크 아님, 같은 스코프 머신 한 칸 확장(계정>프로젝트>에이전트 위에 조직/공유 스코프).

설계 못박음:
- **적층 ≠ 변환**: 개인 여권은 사용자 소유로 유지, 그 위에 별도 공유 메모리 객체(자체 소유·멤버십)를 얹음. 인출 시 개인+공유 합성. Personal Memory Passport 앵커 보존.
- **공유 스코프 = 선택>저장의 무대**: N명 기여 + 공유 캐논 + 요청자 개인기억에서 (요청자·프로젝트·에이전트·프롬프트)별 정밀 선택. 공유 폴더/노션식은 구조적 불가 → 여기서 [[mco-selection-over-storage]] 해자가 반박 불가. 단 "공유 폴더"로 구현하면 도로 인컴번트 요새.
- **공유 write-back = 거버넌스**: 누가 공유 캐논 갱신을 승인? 멤버 간 모순·권한·provenance → 스몰 모델 역할 ③(충돌 감지)가 주역. 나이브 공유 폴더와 갈리는 지점.
- **가드레일**: "프로젝트 공유 스코프"지 "팀 에이전트/협업툴" 아님 — 오케스트레이션·관측 가드레일 연장([[mco-fleet-reframe]]). 스코프가 일하는 게 아니라 멤버 에이전트가 붙어 읽음. 정본 용어 "프로젝트 에이전트" vs "프로젝트 공유 스코프" 미확정.
- **GTM = 랜드앤익스팬드**: 개인 스코프 = 프로슈머 바텀업 획득(경로 A), 공유 스코프 = 팀/엔터프라이즈 확장 매출 + [[mco-manufacturing-positioning]](제조 = 조직 단위)의 자연 착지점.

**미결:** 공유 메모리 소유 모델 — (i) 기여자 연합(개인 소유 유지·위임; 여권 앵커 매끄러움, 거버넌스 복잡) vs (ii) 프로젝트/조직 소유(엔터프라이즈 판매 깔끔, '사용자 소유' 서사와 긴장). 미정.

**How to apply:** MVP 스펙은 개인 스코프로 진행. 팀/공유는 로드맵 상위 스코프로 명시만, 코어 승격은 소유 모델 정한 뒤 차터 반영. 컨셉 코어 건드리면 [[mco-concept-core]] 갱신.
