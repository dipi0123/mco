2026-07-11 세션 판정 (근거 정본: `docs/리서치/MCO_사업성분석_v0_1.md`):

- **아키텍처 커모디티 판정**: md+벡터DB+관계형+단기/장기+스코프태그 = 전 요소 복수 업체 보유, 1인 개발자가 동일 조합 무료 복제(Memory OS, 2026-05). "이 스택이라 경쟁력 있다"는 전제 **기각**.
- **경쟁력의 실제 위치**: 희소/부재 목록 = 강제되는 에이전트별 권한 · bi-temporal 유효기간 · 일급 TTL · 리뷰형 write-back · **베이스라인+10점 넘는 선택 정밀도(아무도 못함)**. 즉 차터의 비저장 요소들이 정확히 승부처. + 복제 불가 자산 = 사용자별 라벨 축적(인박스 판정·승인/기각 = 평가셋 복리).
- **아키텍처 우위 ≠ 승리 (실증)**: mem0(시간 모델링 최약)가 채택 1위, Zep(최고 아키텍처) OSS 중단, Letta(최심층) 피벗, 벤치마크 오염·게이밍 실증. 해자의 실제 출처 = 유통·기본값·축적 데이터·컴플라이언스.
- **개인 'GB 저장' 구독 기각**: MemoryPlugin 3,800명 정체 / Limitless ARR $2.0M→Meta 흡수·무료화 / Supermemory 엔터프라이즈 피벗 / 매출 공개 메모리 회사 0곳. 기각 대상은 구독이 아니라 **GB 가치 미터**(매출∝저장량이면 축적 최적화→노이즈→품질 논거 자충). 방향만 기록(pricing 유보): 획득=무료+출력 순간 / 전환=무료가 못 주는 것(횡단 동기화·벽·팀·정밀).
- **빅테크**: 전원 담장 안 강화, 벤더 횡단 공백 지속(차터 빈틈 판정 유지). AAIF/MCP 로드맵에 메모리 없음, W3C CG 제안 단계 = 표준 창구 열림.
- **한국**: 카카오벤처스→Aristo 'Membase'(벤더 독립 메모리 레이어) 프리시드 2026-06-25 — 국내도 빈 땅 아님.
- **전략 경로**: A 슬롯 정밀도 승부 / B 아웃컴 앱(서류 컴포저 — 지불 실측 유일, 수동 큐레이션 시작 가능) / C 엔터프라이즈(지금 아님). **A·B 첫 실험 병렬**(둘 다 ~공짜), 빌드 순서는 결과로.
- **킬 트리거(분기 감시)**: ① 정밀도 실험 베이스라인+10점 실패 ② OpenAI Memory API 출시·개방 ③ Nous 메모리 자체 번들.

Why: "확실한 경쟁력 없으면 빠른 피벗" 판단 요청에 대한 사실 기반 답 — 인프라에서 확실성은 사전 구매 불가, 살 수 있는 건 싼 반증 실험+킬 트리거.

How to apply: 모든 스펙·피치에서 경쟁력 주장을 저장 스택이 아니라 선택·게이트·벽·라벨 축적에 건다. 분기마다 킬 트리거 점검. [[mco-hermes-verdict]] [[mco-architecture-v01]] [[mco-thinking-order]]
